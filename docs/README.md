Documentação de arquitetura, decisões e convenções. Nada aqui gera runtime — é conhecimento.

## Índice

- [stack.md](./stack.md) — Stack e justificativas.
- [conventions.md](./conventions.md) — Código, naming, imports, commits.
- [data-model.md](./data-model.md) — Tabelas, relações, RLS.
- [auth.md](./auth.md) — Autenticação, sessão, papéis.
- [modules.md](./modules.md) — Catálogo de módulos de domínio.
- [deployment.md](./deployment.md) — Vercel, Supabase, env, CI/CD.
- [roadmap.md](./roadmap.md) — Fases de entrega do MVP.

## O que vai aqui

- ADRs (Architecture Decision Records) quando houver escolha não-óbvia.
- Diagramas (mermaid inline).
- Guias de onboarding para novos devs.

## O que NÃO vai aqui

- Documentação de componentes individuais — mora no `README.md` do módulo/pasta.
- Changelog — `CHANGELOG.md` na raiz.
- Runbooks operacionais — pasta `ops/` (futuro).
